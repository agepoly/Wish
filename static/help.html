What does Wish do?

Aim of the algorithm

The algorithm takes as input a matrix whose lines are the users and whose columns are the slots.

The matrix is filled in with non-negative integers describing the wishes of the users, which we can call “grades”. A small number means that the user would be very satisfied if put in this slot, and a high number means that the user would be disappointed if put in this slot. For a given arrangement of the users in the slots, one can determine a “penalty”: it is the sum of the squares of the “grades” that each user has considering in which slot he/she is placed.

A maximum and minimum number of users is associated to each slot. The aim of the algorithm is to minimize the penalty under these constraints.

Ensuring fair choices

Rules to ensure fair choices

To ensure that everyone has a fair chance of having his/her favorite slot, some rules must be imposed on the choices of the users. Indeed, one could manipulate the results by setting all slots to “hated” but one. A simple solution to this problem is to impose that the user strictly orders all the slots. However, this can be uncomfortable from the user’s point of view, and besides no default value can replace a user that choose not to set his/her wishes.

Therefore, another solution was found. Assume there are n slots. Then, the gradation from wanted to hated is 0, 1, 2,...n-1. Then the rule goes as follows:

  • Up to 1 slot can be hated (grade n-1)
  • Up to 2 slots can be assigned grades greater or equal to n-2.
  • Up to 3 slots can be assigned grades greater or equal to n-3.
  • ….
  • Up to n slots can be assigned grades greater or equal to 0.

Therefore, the default setting of the wishes is to have grade 0 for each slots. This corresponds to somebody having no preferences.

User’s wish page

When the users are setting their wishes, the sliders automatically prevent unfair wishes.

Users with special constraints

Initially this algorithm was intended to put people in slots for an oral exam. In such a case, it can be that some people simply mustn’t be put in some slots because they have another exam at this time. In such a case, it’s nice to be able to avoid the problem by putting a very high grade to the slots that the person mustn’t be put in.

Therefore, the administrator has special rights to access the user’s wish page and select some slots as “avoided” if he/she judges that the user has sufficient reasons to not be put is this slot (see as well Administrate an event).

Private url

To ensure that the users don’t change their wishes to trick the system, it’s good if they don’t know what the preferences of the other users are. Therefore, a unique key is randomly generated for each user. An url is then created using this key, which gives to the user a private access to his/her wish page.

What Wish doesn’t do...

Wish isn’t Doodle

The aim of the algorithm is to arrange people in various slots. It’s not the aim to put all people in the same slot.

Each user is in only one slot in the end

It’s not that easy to modify the algorithm to put each person in two slots.

Recommendations of use

Tips and tricks

If you are not sure how to use some features on the pages of the wish website, try keeping the mouse on this feature. In most cases, a “tip” can appear informing you about how the feature works.

Set your wishes

Before setting your wishes, you might want to take a look at the Rules to ensure fair choices above.

Create an event

Number of slots

If you give too many slots, it will become a bit inconvenient for the users to set their wishes. If you are organizing an oral exam with a lot of students, a good solution is to gather three passage times in one slot and fit three people in the slot.

Deadline

The deadline will be on midnight of the selected date. The algorithm will run as soon as the deadline is passed and the administrator will receive a mail informing him/her that the results are available. He or she will have to go to the admin page to send an email to the participants telling them that the results are available.

Mailing list

The mailing list should contain the e-mail of each of the participants to the event. If the administrator intends to be put in a slot to, his/her mail should be on the list too.

This list will be used to send the following mails:

  • Invitation mail: provides the user with an access to his/her wish page.
  • Update mail: if the administrator changes something in the event organization (slots/deadline), it is possible to send an update mail to warn the users.
  • Reminder mail: the adminisator can send a reminder to peoples that have not set their wishes.
  • Results mail: when the results have been computed, the administrator can inform the users.

Administer an event

Activate the event and send the invitation mails

Once an event has been created, the administrator receives an e-mail containing a link to the admin page. The first time that this page is opened, it sends the invitation mails to the participants.

Administration of a user page and special admin rights

If for some reasons a given user must not be in some of the slots, the administrator can make sure that it won’t happen. To do so, click on the admin link to the user’s page and check the “avoided” case in the corresponding slots.

This assigns a very high grade to this slot in the user’s wish list and makes it therefore highly unlikely for the user to be in this slot. As a consequence, the choices of the user are made unfair, and therefore the user shouldn’t try to modify his/her wish after this.

It is therefore recommended to perform this action shortly before the deadline. It can as well be performed after the deadline, in which case the results have to be computed again using the "Set and Recompute" button on the administrator page.

Change the number of slots

Changing the number of slots is allowed but the administrator should be aware that this can make the choices of the users unfair.

Change the deadline

In case the results have already been computed, changing the deadline suppresses them. They are computed again when the new deadline is over.
